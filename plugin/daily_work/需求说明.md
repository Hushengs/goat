# Chrome 插件需求说明书

## 1. 项目名称
**每周事项倒计时与完成统计插件**

## 2. 项目背景
为用户提供一个可在 Chrome 浏览器中使用的轻量级事项管理插件。用户可以按照星期维度配置每天的固定事项及其完成时间点，并在当天查看每个事项的倒计时、完成状态和整体完成情况，同时支持查看历史日期的记录，便于日常打卡、习惯养成和任务追踪。

## 3. 项目目标
开发一个 Chrome 插件，实现以下目标：

1. 支持按周一至周日分别配置不同事项。
2. 支持为每个事项设置具体完成时间，如 `15:30`。
3. 实时展示每个事项距离目标时间的倒计时。
4. 支持勾选事项的完成/未完成状态。
5. 自动统计当天事项总数、已完成数、未完成数。
6. 支持查看历史日期的事项记录与完成情况。
7. 提供简洁直观、易操作的插件界面。

## 4. 用户角色
### 4.1 普通用户
使用插件进行每日事项配置、打卡、查看统计和历史记录。

## 5. 使用场景
### 5.1 每周固定事项配置
用户希望提前设置不同星期的固定事项，例如：

- 周一：打卡，完成时间 `15:30`
- 周二：写周报，完成时间 `18:00`
- 周三：健身，完成时间 `20:00`

### 5.2 当日执行与打卡
用户在当天打开插件时，可以看到当天所有事项及倒计时，并对已完成事项进行勾选。

### 5.3 每日完成情况统计
用户希望快速知道今天一共有多少件事，完成了多少件，还有多少件未完成。

### 5.4 历史回顾
用户希望查看过去某一天的事项记录与完成状态，了解自己的执行情况。

## 6. 功能需求

## 6.1 周计划配置
### 功能描述
用户可按星期配置每日固定事项模板。

### 功能要求
1. 支持周一到周日分别维护事项列表。
2. 每个星期可添加多条事项。
3. 每条事项至少包含以下字段：
   - 事项名称
   - 完成时间点
4. 支持对事项进行新增、编辑、删除。
5. 支持调整事项排序。
6. 用户保存后，系统按星期生成每日待办基础模板。

### 示例
- 周一：
  - 打卡 - `15:30`
  - 写日报 - `18:00`
- 周五：
  - 周总结 - `17:30`

---

## 6.2 当日事项展示
### 功能描述
插件首页默认展示当天对应星期的事项。

### 功能要求
1. 自动识别当前日期和星期。
2. 加载该星期预先配置的事项模板。
3. 展示每条事项的：
   - 事项名称
   - 目标完成时间
   - 当前状态
   - 倒计时
4. 若当天无配置事项，提示“今日暂无事项”。

## 6.3 倒计时展示
### 功能描述
每个事项需要显示距离目标时间的剩余时间。

### 功能要求
1. 倒计时格式建议为：`时:分:秒`。
2. 当当前时间早于目标时间时，显示剩余倒计时。
3. 当当前时间晚于目标时间时，可显示：
   - `已超时`
   - 或显示超时多久，例如 `已超时 01:15:20`
4. 倒计时需实时刷新，建议每秒更新一次。
5. 对于已完成事项，倒计时可停止或显示“已完成”。

## 6.4 事项完成状态勾选
### 功能描述
用户可手动标记事项是否完成。

### 功能要求
1. 每条事项提供复选框或切换按钮。
2. 状态包含：
   - 未完成
   - 已完成
3. 用户勾选后状态立即保存。
4. 再次取消勾选时，状态恢复为未完成。
5. 状态应按具体日期保存，不影响每周模板本身。

### 说明
例如：周一的“打卡”是模板事项，`2026-06-15` 这一天勾选完成后，只记录该日期已完成，不改变下一个周一的默认状态。

## 6.5 当日统计
### 功能描述
系统自动统计当天事项完成情况。

### 功能要求
页面需展示以下统计信息：
1. 今日事项总数
2. 已完成数量
3. 未完成数量

### 统计规则
- 总数 = 当天全部事项数
- 已完成数 = 状态为已完成的事项数
- 未完成数 = 总数 - 已完成数

### 展示建议
可在首页顶部或底部固定展示摘要，例如：
- 今日共 `5` 件
- 已完成 `3` 件
- 未完成 `2` 件

## 6.6 历史记录查看
### 功能描述
用户可查看历史日期的事项记录。

### 功能要求
1. 支持选择某个历史日期进行查看。
2. 查看内容包括：
   - 当天日期
   - 星期信息
   - 当天事项列表
   - 每项完成状态
   - 每项目标时间
   - 当天统计结果
3. 历史记录应保留用户当时的勾选状态。
4. 如果某一天没有记录，提示“该日期暂无记录”。

### 可选增强
1. 支持按日期列表查看最近 7 天/30 天记录。
2. 支持按完成率排序或筛选。

## 6.7 数据存储
### 功能描述
插件需保存用户配置和每日执行记录。

### 功能要求
1. 保存周计划模板数据。
2. 保存每日事项执行状态数据。
3. 保存历史日期记录数据。
4. 插件关闭或浏览器重启后数据仍可保留。
5. 建议使用 Chrome 插件本地存储能力进行持久化。

## 6.8 插件界面
### 功能描述
提供简洁、清晰的操作界面。

### 页面建议
1. **首页**
   - 显示今日事项
   - 显示每项倒计时
   - 可勾选完成状态
   - 显示今日统计

2. **配置页**
   - 配置周一到周日事项
   - 新增/编辑/删除事项
   - 设置完成时间

3. **历史页**
   - 查看历史日期记录
   - 查看当天统计与完成情况

### UI 要求
1. 界面简洁，适合插件弹窗展示。
2. 重要信息突出显示，如倒计时、已完成数、未完成数。
3. 支持基础响应式布局，适配插件弹窗尺寸。

## 7. 数据结构建议

```json

  "monday": [

  ],
  "tuesday": [],
  "wednesday": [],
  "thursday": [],
  "friday": [],
  "saturday": [],
  "sunday": []

```

### 7.2 每日记录数据
```json
{
  "2026-06-16": {
    "weekday": "tuesday",
    "items": [
      {
        "id": "1",
        "title": "打卡",
        "time": "15:30",
        "status": "done"
      },
      {
        "id": "2",

      "pending": 1

  }
```

## 8. 业务规则
1. 每个事项必须配置事项名称和完成时间。
2. 同一天可存在多个事项。
3. 同一事项模板可在不同日期重复生成记录。
4. 每日记录与周模板相互独立。
5. 当用户首次打开某一天页面时，若该日期尚无记录，则根据对应星期模板自动生成当天记录。
6. 用户修改当天事项状态后，应立即更新统计数据。
7. 历史日期数据生成后，不因后续修改周模板而自动改变。

## 9. 非功能需求
### 9.1 性能要求
1. 插件打开后应快速加载，建议首屏展示时间不超过 2 秒。
2. 倒计时刷新流畅，无明显卡顿。

### 9.2 易用性要求
1. 操作路径简单，用户 3 步内可完成事项新增或勾选。
2. 时间设置方式清晰直观，建议使用时间选择控件。

### 9.3 可靠性要求
1. 数据保存稳定，不因刷新或重启浏览器丢失。
2. 异常情况下应尽量保留用户已保存的数据。

### 9.4 兼容性要求
1. 支持主流 Chrome 浏览器版本。
2. 插件适配 Chrome Extension 标准规范。

## 10. MVP 范围
首版建议实现以下最小可用功能：

1. 周一到周日事项配置
2. 每项配置完成时间
3. 首页显示当天事项与倒计时
4. 勾选完成/未完成状态
5. 今日统计
6. 历史日期记录查看
7. 本地数据持久化

## 11. 可选扩展功能
1. 超时提醒通知
2. 每日完成率统计图表
3. 导出历史记录
4. 支持事项颜色标签分类
5. 支持搜索历史事项
6. 支持云同步账号功能
7. 支持重复规则扩展，如“工作日”或“每月某日”

## 12. 验收标准
### 功能验收
1. 用户可以成功配置每周不同日期的事项和时间。
2. 打开插件后，可以正确显示当天对应事项。
3. 每条事项都能显示实时倒计时。
4. 用户可以勾选已完成/未完成状态，且状态可保存。
5. 页面可以正确统计今日总数、已完成数、未完成数。
6. 用户可以选择历史日期查看当日记录。
7. 浏览器重启后，数据仍能正常读取。

### 体验验收
1. 页面布局清晰，信息易读。
2. 操作流畅，适合日常高频使用。
3. 无明显显示错误或数据错乱。

## 13. 开发建议
### 技术方向建议
1. 使用 `Chrome Extension Manifest V3`
2. 前端界面可采用：
   - 原生 HTML/CSS/JavaScript
   - 或 `React + TypeScript`
3. 数据存储建议使用：
   - `chrome.storage.local`

### 模块划分建议
1. 配置管理模块
2. 今日事项模块
3. 倒计时模块
4. 状态管理与统计模块
5. 历史记录模块
6. 本地存储模块

## 14. 结论
该插件定位为一个基于星期模板的轻量级日常事项管理工具，核心价值在于“按周配置 + 当日倒计时 + 完成统计 + 历史回看”。首版应优先保证配置、展示、勾选、统计和历史记录等核心流程稳定可用，再逐步扩展提醒、图表和同步等高级能力。
