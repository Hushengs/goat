# 每周事项倒计时插件 UI 设计文档

## 1. 文档目标
基于 `需求说明.md`，输出一份适用于 Chrome 插件弹窗场景的简洁 UI 设计方案，用于指导页面布局、组件设计和交互实现。

## 2. 设计原则
- 简洁优先：在有限弹窗空间内突出任务、时间、状态、统计。
- 高效操作：用户应可快速查看今日事项，并在 1 到 3 步内完成打卡或配置。
- 信息分层清晰：优先展示今日最重要信息，次级功能通过分页或标签切换承载。
- 状态明确：完成、未完成、超时、空状态要一眼可辨。

## 3. 整体信息架构
插件采用三页结构，适合 Popup 弹窗使用：

1. `今日`：查看当天事项、倒计时、完成状态、统计摘要。
2. `配置`：按周一到周日维护固定事项模板。
3. `历史`：查看指定日期的事项记录与统计。

建议使用顶部标题栏 + 页签切换的结构，默认进入 `今日` 页。

## 4. 弹窗尺寸与布局建议
## 4.1 尺寸建议
- 弹窗宽度：`360px`
- 弹窗高度：`560px`
- 页面安全边距：`16px`
- 卡片圆角：`12px`
- 主内容区支持纵向滚动

## 4.2 布局骨架
- 顶部：插件标题 + 日期/星期信息
- 中部：当前页面核心内容
- 底部或顶部页签区：`今日 / 配置 / 历史`

建议结构如下：

```text
+----------------------------------+
| 标题 / 日期信息                  |
| 页签: 今日 配置 历史             |
|----------------------------------|
| 页面内容区                       |
|                                  |
|                                  |
|                                  |
+----------------------------------+
```

## 5. 视觉风格建议
- 风格：轻量、清爽、偏工具型
- 主色：蓝色系，如 `#2563EB`
- 成功色：绿色，如 `#16A34A`
- 警示色：橙红色，如 `#EA580C`
- 背景色：浅灰白，如 `#F8FAFC`
- 主文本：深灰，如 `#0F172A`
- 次文本：中灰，如 `#64748B`

字体层级建议：
- 页面标题：`18px / Semibold`
- 区块标题：`14px / Semibold`
- 正文：`13px / Regular`
- 辅助信息：`12px / Regular`
- 倒计时数字：`16px / Bold`

## 6. 页面设计
## 6.1 今日页
### 页面目标
让用户打开插件后，第一时间看到今天要做什么、何时截止、还差多少、完成了多少。

### 页面结构
1. 顶部日期卡片
   - 显示今天日期
   - 显示星期信息
   - 可附带一句摘要，如“今天共 5 项待完成”

2. 今日统计卡片
   - 今日总数
   - 已完成数
   - 未完成数

3. 事项列表区
   - 按卡片列表展示今日事项
   - 每项展示：
     - 事项名称
     - 目标时间
     - 倒计时或超时状态
     - 完成勾选控件

### 单条事项卡片设计
- 左侧：完成复选框
- 中间：事项名称 + 目标时间
- 右侧：倒计时状态

建议样式：
- 未完成：白底卡片
- 已完成：浅绿色背景或勾选后降低强调
- 已超时：倒计时文字切换为警示色

建议文案示例：
- `剩余 02:15:08`
- `已超时 00:45:10`
- `已完成`

### 空状态
当天无事项时显示：
- 主文案：`今日暂无事项`
- 次文案：`可前往配置页添加每周固定事项`
- 按钮：`去配置`

## 6.2 配置页
### 页面目标
帮助用户按星期维护固定事项模板。

### 页面结构
1. 星期切换栏
   - 周一到周日横向切换
   - 当前选中星期高亮

2. 事项配置列表
   - 每条事项显示：
     - 事项名称输入框
     - 时间选择控件
     - 拖拽排序入口或上下移动按钮
     - 编辑/删除操作

3. 底部主操作区
   - `新增事项`
   - `保存配置`

### 交互建议
- 新增事项：点击后在列表底部插入空白事项行
- 编辑事项：支持直接行内编辑
- 删除事项：点击删除图标后弹出轻确认
- 时间设置：使用原生时间选择器，减少输入成本
- 保存后反馈：显示轻提示，如 `已保存`

### 空状态
某星期尚未配置事项时显示：
- 主文案：`当前星期暂无事项`
- 操作按钮：`新增第一项`

## 6.3 历史页
### 页面目标
帮助用户查看过去某一天的执行结果。

### 页面结构
1. 日期选择区
   - 日期选择器
   - 可附加快捷入口：`最近 7 天`

2. 历史统计卡片
   - 当天总数
   - 已完成数
   - 未完成数

3. 历史事项列表
   - 展示事项名称
   - 目标时间
   - 完成状态

### 历史页规则
- 历史页默认只读，不允许修改模板
- 如需允许改历史状态，应单独加“编辑记录”能力；MVP 建议只读展示

### 空状态
若该日期暂无记录，显示：
- 主文案：`该日期暂无记录`
- 次文案：`当天可能没有生成事项或尚未打开过插件`

## 7. 核心组件说明
## 7.1 顶部栏
- 左侧：插件名称
- 右侧：当前日期或简单图标入口
- 保持固定，便于稳定识别当前页面

## 7.2 页签组件
- 三个页签：`今日`、`配置`、`历史`
- 当前页签高亮，并有底部指示条
- 点击后切换页面内容，不跳出弹窗

## 7.3 统计卡片
- 建议使用三列布局
- 每列展示数字 + 标签
- 数字视觉权重高于标签

示例：
- `5` 今日总数
- `3` 已完成
- `2` 未完成

## 7.4 事项卡片
- 支持 hover 高亮
- 已完成状态要有明显反馈
- 时间与状态信息分开显示，避免信息拥挤

## 7.5 提示反馈
- 保存成功：轻提示 Toast
- 删除确认：轻量确认弹层
- 数据为空：插图可选，优先用文案保持轻量

## 8. 状态与交互规范
## 8.1 倒计时状态
- 未到时间：显示剩余时间
- 已超时：显示超时文案和超时时长
- 已完成：显示 `已完成`，停止倒计时或弱化显示

## 8.2 完成状态切换
- 默认未完成
- 用户勾选后立即保存
- 勾选成功后同步刷新统计数据

## 8.3 数据反馈
- 首次进入某日且无记录时，自动根据周模板生成当日数据
- 保存操作要有即时反馈
- 异常情况展示简短错误提示，如 `保存失败，请重试`

## 9. 响应与可用性建议
- 所有点击区域高度不低于 `36px`
- 列表项间距建议 `8px` 到 `12px`
- 长事项名称超出时单行省略，必要时支持换行
- 配置页操作按钮应固定在底部或始终可见，减少滚动成本

## 10. MVP UI 范围
首版建议优先实现以下界面能力：

1. 今日页完整可用
2. 配置页支持新增、编辑、删除、保存
3. 历史页支持按日期查看记录
4. 倒计时、完成状态、统计结果可实时更新
5. 基于本地存储保持数据持久化

## 11. 推荐落地顺序
1. 先完成 `今日页` 布局与事项卡片
2. 再完成 `配置页` 的星期切换与表单编辑
3. 最后补齐 `历史页` 与空状态、提示反馈

## 12. 结论
该插件 UI 应围绕“今日执行效率”展开，默认突出今日事项、倒计时和完成统计；配置与历史作为辅助页面承载管理与回顾功能。整体界面应保持轻量、清晰、可快速操作，适配 Chrome 插件弹窗的小尺寸使用场景。
