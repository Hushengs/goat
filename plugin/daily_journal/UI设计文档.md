# 每日工作记录与周汇总插件 UI 设计文档

## 1. 文档目标
基于需求说明，输出一份适用于 Chrome 插件弹窗场景的简洁 UI 设计方案，用于指导页面结构、核心交互和视觉呈现。

## 2. 设计原则
1. 高效率录入：优先保证“快速新增工作记录”。
2. 信息清晰：当日记录、工时统计、周汇总内容一眼可读。
3. 操作轻量：控制页面层级，减少跳转和重复输入。
4. 适配弹窗：界面适合 Chrome Extension 弹窗尺寸，避免复杂布局。

## 3. 设计范围
本次 UI 设计覆盖以下 4 个页面：
1. 首页 / 今日记录页
2. 项目管理页
3. 历史记录页
4. 周汇总页

## 4. 弹窗规格建议
- 弹窗宽度：`380px` 到 `420px`
- 弹窗高度：`560px` 到 `680px`
- 页面结构：顶部导航 + 主内容区 + 底部操作区（按需）
- 内容滚动：仅主内容区滚动，顶部导航保持固定

## 5. 信息架构
```text
顶部区域
- 插件标题
- 页面切换导航

页面内容
- 今日记录
- 项目管理
- 历史记录
- 周汇总
```

建议使用顶部 `Tab` 或分段导航切换四个页面，减少多层返回操作。

## 6. 页面设计

### 6.1 首页 / 今日记录页
#### 页面目标
让用户在最少步骤内完成当天工作登记，并快速查看当日总工时。

#### 页面结构
1. 顶部信息区
   - 页面标题：`今日记录`
   - 日期选择器，默认当天
   - 当日工时合计
2. 新增记录区
   - 项目下拉框
   - 工作事项多行输入框
   - 工时输入框
   - `保存` 按钮
3. 记录列表区
   - 展示当天全部记录
   - 每条记录支持编辑、删除
4. 空状态区
   - 当天无记录时提示 `当天暂无工作记录`

#### 单条记录卡片建议
- 第一行：项目名称
- 第二行：事项内容
- 第三行：工时 + 编辑按钮 + 删除按钮

#### 交互说明
1. 打开插件默认进入当天记录。
2. 用户先选择项目，再填写事项和工时。
3. 点击 `保存` 后，新记录立即插入列表顶部或底部。
4. 若没有可选项目，表单区域显示提示并引导进入项目管理页。

### 6.2 项目管理页
#### 页面目标
维护工作项目，为每日登记提供稳定的项目选项。

#### 页面结构
1. 顶部说明区
   - 页面标题：`项目管理`
   - 简短提示：`停用项目不会出现在登记下拉框中`
2. 项目列表区
   - 项目名称
   - 负责人/标识信息
   - 排序号
   - 启用状态
   - 编辑/删除操作
3. 新增或编辑表单区
   - 项目名称输入框
   - 负责人输入框
   - 排序号输入框
   - 启用开关
   - `保存项目` 按钮

#### 交互说明
1. 支持新增、编辑、删除、启用/停用。
2. 长项目名称默认两行显示，超出部分省略。
3. 排序结果同时影响首页下拉顺序与周汇总项目顺序。

### 6.3 历史记录页
#### 页面目标
按日期回看历史工作内容，并支持直接修正历史记录。

#### 页面结构
1. 顶部筛选区
   - 页面标题：`历史记录`
   - 日期选择器
   - 快捷筛选：`最近7天`、`最近30天`（可选）
2. 统计摘要区
   - 当前日期
   - 当日工时合计
3. 历史列表区
   - 展示该日期的全部记录
   - 支持编辑、删除
4. 空状态区
   - 提示 `该日期暂无记录`

#### 交互说明
1. 切换日期后，列表和工时统计同步刷新。
2. 历史页保留与首页一致的记录卡片样式，降低学习成本。

### 6.4 周汇总页
#### 页面目标
清晰展示本周按项目分组的工作汇总，并支持一键复制。

#### 页面结构
1. 顶部信息区
   - 页面标题：`周汇总`
   - 周范围显示，如 `2026-06-15 ~ 2026-06-21`
   - 本周总工时
2. 汇总操作区
   - `生成汇总` 按钮
   - `复制汇总` 按钮
3. 汇总预览区
   - 多行文本展示最终内容
   - 按项目分组
   - 展示项目小计和总工时
4. 空状态区
   - 提示 `本周暂无工作记录`

#### 交互说明
1. 进入页面后可自动生成当前周汇总，或通过按钮手动刷新。
2. 复制成功后提示 `已复制`。
3. 预览区需支持滚动和全文选中，便于人工检查。

## 7. 关键组件设计

### 7.1 顶部导航
- 建议使用 `Tab`：`记录`、`项目`、`历史`、`汇总`
- 当前页高亮，未选中状态保持弱化

### 7.2 表单组件
- 项目选择：下拉框
- 工作事项：多行文本框，支持较长中文输入
- 工时：短输入框，右侧固定显示 `H`
- 日期：原生日历选择器

### 7.3 列表卡片
- 使用卡片式或分组列表式
- 保持统一边距、圆角和层次
- 操作按钮建议使用图标 + 文本或简洁图标

### 7.4 提示反馈
- 保存成功：短暂提示
- 复制成功：短暂提示
- 校验失败：字段下方提示错误原因
- 删除操作：建议二次确认

## 8. 状态设计
1. 空项目状态：提示先配置项目。
2. 空记录状态：提示当天暂无工作记录。
3. 空周汇总状态：提示本周暂无工作记录。
4. 加载状态：列表或汇总区域显示轻量加载提示。
5. 错误状态：保存失败、复制失败时展示明确原因。

## 9. 视觉风格建议
- 整体风格：简洁、轻量、偏工具型
- 主色：建议使用蓝色或绿色作为主强调色
- 背景色：浅灰或白色
- 卡片层级：通过边框或浅阴影区分
- 字号建议：
  - 页面标题：`16px`
  - 区块标题：`14px`
  - 正文：`12px` 到 `13px`
  - 辅助信息：`12px`

## 10. 交互优先级建议
MVP 阶段建议优先突出以下能力：
1. 首页快速新增记录
2. 当日工时统计可见
3. 项目管理简单可维护
4. 周汇总预览与复制稳定可用

## 11. 推荐页面流转
```text
首次使用
首页发现无项目
-> 进入项目管理页配置项目
-> 返回首页新增工作记录
-> 切换历史页查看过往记录
-> 切换周汇总页生成并复制内容
```

## 12. 实现备注
1. 首页、历史页尽量复用同一套记录列表组件。
2. 周汇总页建议使用只读文本区域展示结果，保证复制稳定。
3. 项目名称可能较长，所有列表需提前考虑换行和截断策略。
4. 所有核心操作应尽量在弹窗内完成，不依赖额外页面。
