# A 股智能选股谷歌插件需求说明

## 1. 项目背景

开发一个基于 Chrome/Edge 浏览器的股票分析插件，面向关注 A 股市场的个人投资者。用户可维护一个自选资源池，插件基于资源池内股票的近期走势、价格、成交量等数据，计算每只股票当日上涨和下跌的概率，并结合持仓数据给出当日最值得买入和最值得卖出的股票建议，同时展示当日盈亏情况。

## 2. 产品目标

### 2.1 核心目标

1. 让用户能够快速维护自己的 A 股股票资源池。
2. 自动拉取资源池股票的行情与技术指标数据。
3. 对资源池股票进行当日上涨/下跌概率评估。
4. 每个交易日输出 1 只最适合买入的股票与建议买入价。
5. 每个交易日输出 1 只最适合卖出的股票与建议卖出价。
6. 结合用户持仓，计算并展示当日盈亏。

### 2.2 产品定位

该插件定位为“轻量级交易辅助决策工具”，用于提供数据驱动的参考建议，不直接替代用户做最终交易决策。

## 3. 使用对象

1. 有 A 股投资需求的个人用户。
2. 已有自选股票池，需要快速判断当日交易机会的用户。
3. 需要在浏览器中轻量查看持仓收益和买卖建议的用户。

## 4. 术语说明

- `资源池`：用户添加并持续关注的一组 A 股股票列表。
- `持仓`：用户当前已买入且尚未卖出的股票记录。
- `上涨概率`：系统基于规则模型或评分模型估算股票当日上涨的可能性。
- `下跌概率`：系统基于规则模型或评分模型估算股票当日下跌的可能性。
- `建议买入价`：结合支撑位、均线位置、波动区间等给出的参考买点。
- `建议卖出价`：结合压力位、止盈位、止损位等给出的参考卖点。
- `当日盈亏`：持仓股票相对于上一交易日收盘或当日买入成本所形成的盈亏。

## 5. 产品范围

### 5.1 本期范围

1. 资源池管理。
2. A 股基础行情数据展示。
3. 涨跌概率评估。
4. 每日买入/卖出推荐。
5. 持仓录入与当日盈亏计算。
6. 浏览器插件内的可视化展示。

### 5.2 暂不纳入

1. 自动下单交易。
2. 券商账户直连。
3. 港股、美股、基金、期货等其他市场支持。
4. 基于深度学习的复杂量化策略训练。
5. 多账户资产归集。

## 6. 业务流程

### 6.1 主流程

1. 用户安装插件并完成首次初始化。
2. 用户在资源池中添加 A 股股票代码或名称。
3. 系统自动获取股票的最新行情、K 线、成交量及相关技术指标。
4. 系统对资源池内股票进行当日上涨/下跌概率计算。
5. 系统从资源池中选出最适合买入的 1 只股票，并给出建议买入价。
6. 系统从持仓或资源池中选出最适合卖出的 1 只股票，并给出建议卖出价。
7. 用户录入或同步持仓数据，系统计算当日盈亏并展示。

### 6.2 关键决策原则

1. 买入推荐优先从资源池中选取综合评分最高且风险可控的股票。
2. 卖出推荐优先从当前持仓中选取达到止盈、跌破支撑或出现转弱信号的股票。
3. 若无持仓，则卖出推荐模块显示“暂无持仓，无卖出建议”。

## 7. 功能需求

### 7.1 资源池管理

#### 7.1.1 功能说明

用户可以维护一个自选股票资源池，作为分析和推荐的基础范围。

#### 7.1.2 功能点

1. 支持按股票代码添加 A 股股票。
2. 支持按股票名称搜索并添加股票。
3. 支持删除资源池中的股票。
4. 支持编辑备注信息，如行业、关注原因、风险标签。
5. 支持资源池列表展示以下字段：
   - 股票代码
   - 股票名称
   - 最新价
   - 涨跌幅
   - 成交量
   - 成交额
   - 上涨概率
   - 下跌概率
   - 综合评分
6. 支持按涨跌幅、上涨概率、综合评分排序。
7. 支持手动刷新单只或全部股票数据。

#### 7.1.3 业务规则

1. 仅允许添加 A 股合法股票代码。
2. 重复添加同一股票时，系统应提示“该股票已在资源池中”。
3. 资源池默认上限为 100 只，后续可配置扩展。

### 7.2 行情数据获取

#### 7.2.1 数据范围

系统需要为资源池中的每只股票获取以下数据：

1. 实时或准实时价格。
2. 开盘价、最高价、最低价、昨收价。
3. 当日涨跌幅。
4. 成交量、成交额、量比。
5. 最近 5 日、10 日、20 日价格走势。
6. 均线数据，如 MA5、MA10、MA20。
7. 波动指标，如振幅、换手率。

#### 7.2.2 数据来源要求

1. 优先使用稳定、可持续访问的股票数据接口。
2. 若浏览器插件直接访问第三方接口受限，应通过后端代理服务转发请求。
3. 需要考虑交易日、非交易日和午间休市时段的数据更新逻辑。

#### 7.2.3 刷新机制

1. 插件打开时自动拉取最新数据。
2. 交易时间内支持按固定间隔自动刷新，建议 1 至 5 分钟一次。
3. 非交易时间显示最近一次可用数据，并标记“非交易时段”。

### 7.3 涨跌概率评估

#### 7.3.1 目标

根据资源池内股票的近期走势、价格、成交量等指标，估算股票当日上涨和下跌的概率。

#### 7.3.2 评估输入

建议纳入以下维度：

1. 趋势维度：
   - 最近 3 日、5 日、10 日涨跌幅
   - 当前价格与 MA5、MA10、MA20 的关系
2. 量能维度：
   - 当日成交量与近 5 日均量对比
   - 量比
   - 放量上涨或缩量回调特征
3. 波动维度：
   - 振幅
   - 高低点波动区间
4. 强弱维度：
   - 相对强弱分数
   - 是否跑赢所属板块或大盘
5. 风险维度：
   - 连续大涨后的追高风险
   - 高位放量滞涨
   - 跌破关键均线

#### 7.3.3 输出结果

每只股票至少输出以下内容：

1. 上涨概率，取值范围 0% 至 100%。
2. 下跌概率，取值范围 0% 至 100%。
3. 综合评分，取值范围 0 至 100。
4. 风险等级，分为低、中、高。
5. 推荐理由，如“均线多头排列”“缩量回踩支撑”“放量突破”。

#### 7.3.4 基础算法建议

第一期可采用规则评分模型，避免一开始引入高复杂度机器学习方案。

示例评分逻辑如下：

1. 价格站上 MA5、MA10、MA20，各加一定分值。
2. 最近 5 日涨幅适中且未大幅拉升，加分。
3. 当日或近 3 日量能温和放大，加分。
4. 若高位放量滞涨、跌破 MA10 或连续大涨后乖离过大，则减分。
5. 依据总分映射上涨概率与下跌概率。

示例映射规则：

- 综合评分 >= 80：上涨概率 70% 以上。
- 综合评分 60 到 79：上涨概率 55% 至 70%。
- 综合评分 40 到 59：震荡，涨跌概率接近。
- 综合评分 < 40：下跌概率偏高。

说明：最终算法可配置化，后续支持参数调整和策略升级。

### 7.4 买入推荐

#### 7.4.1 目标

每日从资源池中选出 1 只最适合当日买入的股票，并给出建议买入价。

#### 7.4.2 推荐条件

买入候选股票应同时考虑以下条件：

1. 上涨概率高于设定阈值，例如 60%。
2. 综合评分在资源池中排名靠前。
3. 当前股价未明显偏离短期均线，避免过度追高。
4. 量价关系健康，如回踩不破位或放量突破。
5. 风险等级不为高风险。

#### 7.4.3 建议买入价逻辑

建议买入价可按以下优先级计算：

1. 优先取短期支撑位附近价格。
2. 若股价强势突破，可取突破确认价附近作为参考。
3. 若波动较大，可输出买入区间而非单点价格。

建议展示格式：

- 推荐股票：`股票名称（代码）`
- 建议买入价：`xx.xx`
- 可接受买入区间：`xx.xx - xx.xx`
- 推荐理由：`上涨概率高、量价配合良好、靠近支撑位`
- 风险提示：`若跌破 xx.xx 建议观望`

#### 7.4.4 排序规则

当多只股票同时满足条件时，按以下顺序择优：

1. 综合评分高者优先。
2. 上涨概率高者优先。
3. 风险等级低者优先。
4. 买点更接近支撑位者优先。

### 7.5 卖出推荐

#### 7.5.1 目标

每日选出 1 只最适合卖出的股票，并给出建议卖出价。

#### 7.5.2 推荐范围

1. 默认优先从当前持仓中筛选卖出对象。
2. 若用户无持仓，可仅展示资源池中的减仓风险提示，不输出正式卖出建议。

#### 7.5.3 卖出信号

以下情况可作为卖出或减仓信号：

1. 下跌概率高于设定阈值，例如 60%。
2. 达到预设止盈目标。
3. 跌破重要支撑位或 MA10/MA20。
4. 高位放量滞涨。
5. 短线涨幅过大且出现转弱迹象。

#### 7.5.4 建议卖出价逻辑

建议卖出价可按以下方式计算：

1. 接近压力位时给出止盈价。
2. 触发跌破支撑时给出止损价。
3. 对盈利持仓，可同时给出分批卖出价和保本止盈价。

建议展示格式：

- 推荐卖出股票：`股票名称（代码）`
- 建议卖出价：`xx.xx`
- 参考止盈价：`xx.xx`
- 参考止损价：`xx.xx`
- 推荐理由：`下跌概率升高、量价转弱、接近压力位`

### 7.6 持仓管理

#### 7.6.1 功能说明

用户可以录入和维护当前持仓数据，系统据此计算当日盈亏和个股收益表现。

#### 7.6.2 持仓字段

每条持仓至少包含：

1. 股票代码
2. 股票名称
3. 持仓数量
4. 成本价
5. 最新价
6. 昨收价
7. 持仓市值
8. 浮动盈亏
9. 当日盈亏
10. 盈亏比例

#### 7.6.3 持仓操作

1. 支持手动新增持仓。
2. 支持编辑持仓数量和成本价。
3. 支持删除持仓记录。
4. 支持按股票代码自动关联行情数据。

### 7.7 当日盈亏计算

#### 7.7.1 计算目标

基于持仓数据和最新行情，计算用户持仓的当日盈亏与总收益情况。

#### 7.7.2 计算公式

1. 持仓市值 = 最新价 × 持仓数量
2. 浮动盈亏 = (最新价 - 成本价) × 持仓数量
3. 浮动盈亏比例 = (最新价 - 成本价) / 成本价 × 100%
4. 当日盈亏 = (最新价 - 昨收价) × 持仓数量
5. 当日盈亏比例 = (最新价 - 昨收价) / 昨收价 × 100%

#### 7.7.3 汇总展示

系统需展示：

1. 总持仓市值
2. 总浮动盈亏
3. 总当日盈亏
4. 盈利股票数量
5. 亏损股票数量

### 7.8 首页看板

#### 7.8.1 展示内容

插件首页应包含以下核心区域：

1. 今日最适合买入股票卡片。
2. 今日最适合卖出股票卡片。
3. 资源池股票列表。
4. 持仓盈亏概览。
5. 数据更新时间。

#### 7.8.2 交互要求

1. 支持点击股票查看详细分析。
2. 支持一键刷新数据。
3. 支持按标签筛选，如“高概率上涨”“高风险”“持仓中”。

### 7.9 股票详情分析

#### 7.9.1 展示内容

点击股票后进入详情面板，展示：

1. 基础行情信息。
2. 最近 5 日、10 日、20 日走势摘要。
3. 成交量变化趋势。
4. 均线关系说明。
5. 上涨/下跌概率及原因解释。
6. 建议买入价、建议卖出价、止损位、止盈位。

## 8. 非功能需求

### 8.1 性能要求

1. 插件主页面首次打开时间不超过 3 秒。
2. 单次刷新 50 只股票的行情分析结果应在 5 秒内完成。
3. 页面操作响应时间应控制在 1 秒内。

### 8.2 可用性要求

1. 界面简洁，核心结论优先展示。
2. 对非专业用户应尽量使用易理解的术语和风险提示。
3. 关键数据异常时应给出明确提示，如“接口异常”“数据延迟”。

### 8.3 准确性要求

1. 插件输出仅作为辅助决策，不承诺收益或准确率。
2. 所有推荐结果必须展示计算时间与依据摘要。
3. 概率模型结果应支持持续迭代优化。

### 8.4 安全性要求

1. 持仓数据默认保存在本地浏览器存储中。
2. 如有云同步需求，需明确用户授权。
3. 不存储用户券商账号密码等敏感信息。

## 9. 技术方案建议

### 9.1 插件形态

建议基于 Chrome Extension Manifest V3 实现，兼容 Chrome 和 Edge。

### 9.2 前端模块

1. `Popup`：展示今日推荐与盈亏概览。
2. `Options`：配置资源池、持仓、刷新频率、风险参数。
3. `Background`：负责定时刷新、缓存、消息通信。
4. `Storage`：保存资源池、持仓、用户配置和缓存结果。

### 9.3 后端能力建议

若第三方股票数据接口受限，建议增加轻量后端服务，负责：

1. 拉取股票行情数据。
2. 统一清洗与标准化字段。
3. 执行概率评分逻辑。
4. 返回插件可直接使用的推荐结果。

## 11. 异常场景

1. 股票代码无效时，提示用户重新输入。
2. 数据接口异常时，展示降级提示并保留上次缓存数据。
3. 某只股票数据缺失时，不影响其他股票正常分析。
4. 用户未添加资源池时，首页提示先添加股票。
5. 用户未录入持仓时，盈亏与卖出建议区域展示空状态。

## 12. 验收标准

### 12.1 功能验收

1. 用户可以成功添加、删除、编辑资源池股票。
2. 系统可以正常拉取并展示股票基础行情。
3. 系统可以对资源池股票输出上涨概率、下跌概率和综合评分。
4. 系统每日能够选出 1 只最适合买入的股票并输出建议买入价。
5. 系统每日能够选出 1 只最适合卖出的股票并输出建议卖出价。
6. 用户录入持仓后，系统能正确计算个股和总持仓的当日盈亏。

### 12.2 结果验收

1. 推荐结果需包含明确理由与风险提示。
2. 当无足够数据支撑时，系统应明确提示“不建议交易”。
3. 插件关闭后重新打开，资源池和持仓数据不丢失。

## 13. 后续迭代方向

1. 增加板块热度和大盘情绪分析。
2. 支持自定义选股策略参数。
3. 支持消息提醒，如“达到买点”“跌破止损位”。
4. 支持历史回测，验证推荐策略效果。
5. 支持云端同步和多设备使用。
