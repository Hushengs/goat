# A 股智能选股谷歌插件 UI 设计文档

## 1. 文档目标

本文档基于《需求说明》整理插件的界面结构与交互方案，用于指导产品、UI 设计和前端实现。目标是让用户在浏览器插件中快速完成以下任务：

1. 查看今日最值得关注的买入和卖出建议。
2. 浏览资源池股票的核心行情与概率结果。
3. 查看持仓盈亏概览。
4. 维护资源池、持仓和基础策略配置。

## 2. 设计原则

1. 结论优先：先展示买卖建议和盈亏摘要，再展示明细列表。
2. 轻量高效：符合 Chrome/Edge 插件使用习惯，减少复杂跳转。
3. 信息分层：首页只放核心结论，深度分析放到详情面板或配置页。
4. 风险明确：所有推荐都展示理由、风险等级和数据更新时间。
5. 状态可感知：加载中、非交易时段、数据异常、空状态都要明确提示。

## 3. 插件信息架构

插件建议拆分为 3 个主要界面：

1. `Popup 首页`
   - 每次点击插件图标后打开
   - 展示今日推荐、持仓盈亏、资源池摘要
2. `Options 配置页`
   - 管理资源池、持仓、刷新频率、风险阈值
3. `股票详情面板`
   - 从首页列表点击单只股票进入
   - 展示更完整的走势、评分原因和操作建议

## 4. 页面结构总览

### 4.1 Popup 首页

建议尺寸：

- 宽度：`380px - 420px`
- 高度：`560px - 680px`
- 采用纵向滚动布局

页面从上到下分为 6 个区域：

1. 顶部状态栏
2. 今日建议区
3. 持仓盈亏概览
4. 快捷筛选栏
5. 资源池股票列表
6. 底部快捷操作

推荐线框如下：

```text
+--------------------------------------+
| Logo/标题      更新时间     刷新按钮 |
+--------------------------------------+
| 今日买入推荐卡片                     |
| 股票 / 概率 / 买入价 / 理由 / 风险   |
+--------------------------------------+
| 今日卖出推荐卡片                     |
| 股票 / 概率 / 卖出价 / 理由 / 风险   |
+--------------------------------------+
| 持仓盈亏概览                         |
| 总市值 / 浮盈亏 / 当日盈亏 / 涨跌家数|
+--------------------------------------+
| 标签筛选: 全部 高概率上涨 高风险 持仓|
+--------------------------------------+
| 资源池列表                           |
| 股票A                                |
| 股票B                                |
| 股票C                                |
+--------------------------------------+
| 管理资源池 | 管理持仓 | 打开设置     |
+--------------------------------------+
```

### 4.2 Options 配置页

Options 页建议采用标签页结构，避免单页信息过长：

1. `资源池管理`
2. `持仓管理`
3. `策略与刷新设置`
4. `数据与说明`

### 4.3 股票详情面板

详情页可采用新标签页或内嵌抽屉，优先建议使用独立详情页，便于容纳更多信息。结构包括：

1. 股票基础信息
2. 价格与涨跌摘要
3. 走势与均线说明
4. 量能与波动说明
5. 概率与评分解释
6. 买卖价建议
7. 风险提示与数据时间

## 5. 核心界面设计

### 5.1 顶部状态栏

展示内容：

1. 插件名称，如 `A 股智能选股`
2. 当前市场状态：`交易中`、`午间休市`、`非交易时段`
3. 最近更新时间
4. `刷新`按钮
5. `设置`入口

交互要求：

1. 点击 `刷新` 后立即进入加载状态，按钮显示转圈或禁用。
2. 若接口异常，顶部出现轻提示：`数据更新失败，已展示最近缓存结果`。

### 5.2 今日建议区

今日建议区包含两张并列风格一致的卡片：

1. `今日最适合买入`
2. `今日最适合卖出`

单张卡片建议字段：

1. 股票名称 + 代码
2. 核心概率值
3. 建议价格
4. 区间或止盈止损价
5. 2 至 3 条推荐理由
6. 风险等级标签
7. 查看详情入口

卡片设计建议：

1. 买入卡片主色使用红色系强调机会。
2. 卖出卡片主色使用绿色或橙色系强调风险/兑现。
3. 风险等级使用标签色区分：
   - 低风险：绿色
   - 中风险：橙色
   - 高风险：红色

空状态文案：

- 无买入建议：`今日暂无满足条件的买入机会`
- 无卖出建议：`暂无持仓或暂无明确卖出信号`

### 5.3 持仓盈亏概览

该区域以 2 x 2 数据卡形式展示：

1. 总持仓市值
2. 总浮动盈亏
3. 总当日盈亏
4. 盈利 / 亏损股票数量

展示规范：

1. 盈利数值使用红色，亏损数值使用绿色。
2. 金额与百分比同时展示时，金额优先、比例次之。
3. 无持仓时展示空状态卡片，并提供 `新增持仓` 按钮。

### 5.4 快捷筛选栏

建议使用横向可滚动标签：

- 全部
- 高概率上涨
- 高概率下跌
- 高风险
- 持仓中
- 今日推荐

交互要求：

1. 单选切换，默认 `全部`。
2. 切换标签后仅刷新列表区域，不影响上方推荐卡。

### 5.5 资源池股票列表

列表为首页最核心的信息区，建议默认展示 6 至 10 只股票，并支持滚动。

单条列表项包含：

1. 股票名称 + 代码
2. 最新价
3. 涨跌幅
4. 上涨概率 / 下跌概率
5. 综合评分
6. 风险等级
7. 持仓标识
8. 行内操作：`刷新`、`详情`

列表项层级建议：

1. 第一行显示股票名称、代码、最新价、涨跌幅。
2. 第二行显示概率条、评分和风险标签。
3. 若有备注或关注原因，可在详情中查看，不放首页主列表。

视觉建议：

1. 上涨概率和下跌概率用双段进度条或左右对比条显示。
2. 综合评分可用数字 + 小徽标形式展示，如 `82 / 100`。
3. 风险等级标签不超过 1 个，避免信息噪声。

### 5.6 底部快捷操作

建议保留 3 个高频入口：

1. `管理资源池`
2. `管理持仓`
3. `打开设置`

若空间不足，可将 `打开设置` 收纳到右上角菜单。

## 6. Options 配置页设计

### 6.1 资源池管理

页面结构：

1. 顶部添加区
2. 股票列表区
3. 备注编辑区或弹窗

添加区支持：

1. 输入股票代码添加
2. 输入股票名称搜索添加
3. 添加后即时校验是否重复或无效

列表字段建议：

1. 股票代码
2. 股票名称
3. 行业/标签
4. 最新价
5. 上涨概率
6. 综合评分
7. 操作：编辑备注、删除、刷新

### 6.2 持仓管理

页面形式建议为表单 + 表格：

1. 顶部 `新增持仓` 按钮
2. 列表展示持仓数据
3. 支持行内编辑数量与成本价

列表字段：

1. 股票代码
2. 股票名称
3. 持仓数量
4. 成本价
5. 最新价
6. 持仓市值
7. 浮动盈亏
8. 当日盈亏
9. 操作

### 6.3 策略与刷新设置

建议包含以下配置项：

1. 自动刷新开关
2. 刷新频率选择：`1 分钟`、`3 分钟`、`5 分钟`
3. 买入概率阈值
4. 卖出概率阈值
5. 风险等级过滤开关
6. 资源池上限说明

交互建议：

1. 所有配置修改后自动保存到本地。
2. 关键阈值修改后提示：`下次刷新后生效`。

### 6.4 数据与说明

该页用于建立用户信任，建议展示：

1. 数据来源说明
2. 更新时间逻辑
3. 非交易时段说明
4. 风险免责声明

## 7. 股票详情页设计

详情页信息顺序建议如下：

1. 股票头部信息
   - 名称、代码、所属行业、风险等级
2. 实时行情区
   - 最新价、涨跌幅、开高低昨收、成交量、成交额
3. 趋势分析区
   - 最近 5 日/10 日/20 日摘要
   - 与 MA5、MA10、MA20 的关系说明
4. 概率分析区
   - 上涨概率、下跌概率、综合评分
   - 评分理由拆解
5. 操作建议区
   - 建议买入价 / 买入区间
   - 建议卖出价 / 止盈价 / 止损价
6. 风险说明区
   - 例如：`高位放量`、`跌破均线`、`不建议追高`

## 8. 状态设计

### 8.1 加载状态

1. 首页首次打开时，推荐卡片和列表使用骨架屏。
2. 局部刷新单只股票时，仅该条列表项显示加载态。
3. 刷新时保留旧数据，避免页面闪烁。

### 8.2 空状态

1. 未添加资源池：
   - 文案：`还没有自选股票，先添加几只关注标的吧`
   - 按钮：`添加股票`
2. 未录入持仓：
   - 文案：`暂无持仓数据，录入后可查看盈亏与卖出建议`
   - 按钮：`新增持仓`
3. 无推荐结果：
   - 文案：`当前没有足够信号支持推荐，建议继续观察`

### 8.3 异常状态

1. 接口异常：显示顶部错误提示，并保留缓存数据。
2. 个别股票缺失数据：列表中标记 `数据不足`，不阻塞其他股票展示。
3. 非交易时段：页面顶部展示 `非交易时段，以下为最近一次有效数据`。

## 9. 视觉规范建议

### 9.1 色彩

建议采用偏金融工具风格的简洁配色：

1. 主色：深蓝或深灰，体现专业和稳定。
2. 上涨色：红色。
3. 下跌色：绿色。
4. 警示色：橙色。
5. 背景色：浅灰或白色。

### 9.2 字体层级

1. 页面标题：`16px - 18px`
2. 卡片标题：`14px - 16px`
3. 关键价格/概率：`18px - 24px`
4. 辅助说明：`12px - 13px`

### 9.3 组件样式

1. 卡片圆角建议 `10px - 12px`
2. 区块间距建议 `12px - 16px`
3. 标签使用浅底色 + 深文字，减少视觉压迫
4. 概率与评分优先用数字配合轻量图形表达，不做复杂图表

## 10. 文案规范

为了提升易用性，界面文案建议遵循以下原则：

1. 少用纯技术术语，多用用户可理解表达。
2. 推荐理由尽量短句化，如 `均线多头`、`缩量回踩`、`接近压力位`。
3. 风险提示明确可执行，如 `跌破 12.30 建议观望`。
4. 所有结论型区域都附带 `数据时间` 或 `计算时间`。
